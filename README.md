# HH Auto Apply

Инструмент для автоматизации поиска работы на HeadHunter. Скрипт берет на себя рутину: сканирует вакансии по вашим фильтрам, генерирует уникальные сопроводительные письма и автоматически проходит базовые тесты работодателей.

### Как это работает

Вы настраиваете поиск в браузере так, как вам удобно, копируете URL и отдаете его скрипту. Он проходит по списку, имитируя действия человека:
1. Использует ваши куки для авторизации.
2. Подставляет данные в шаблон сопроводительного письма (поддерживается рандомизация фраз для обхода спам-фильтров).
3. Если вакансия требует ответа на тест, скрипт выбирает нейтральные варианты ответов, чтобы пройти формальный барьер.

### Быстрый старт

Для работы понадобятся установленный Python и ваши куки из браузера.

1. **Подготовка окружения:**
   ```bash
   # Создание окружения
   python -m venv .venv

   # Активация (Linux/macOS)
   source .venv/bin/activate

   # Активация (Windows)
   .venv\Scripts\activate

   # Установка зависимостей
   pip install -r requirements.txt
   ```

2. **Авторизация:**
   Установите расширение [Get cookies.txt LOCALLY](https://chromewebstore.google.com/detail/get-cookiestxt-locally/cclelndahbckbenkjhflpdbgdldlbecc), экспортируйте куки с сайта `hh.ru` и сохраните их в файл `cookies.txt` в корне проекта.

3. **Запуск:**
   Скопируйте URL со страницы поиска вакансий и выполните:
   ```bash
   python main.py -u "ВАШ_URL_ИЗ_БРАУЗЕРА"
   ```

### Настройка сопроводительных писем

Текст письма берется из `letter.txt`. Чтобы каждое письмо выглядело уникальным, используйте синтаксис вариантов `{фраза1|фраза2}` и переменную `%vacancyName%`.

**Пример шаблона:**
`{Здравствуйте|Добрый день}! Меня заинтересовала ваша вакансия %vacancyName%. {Буду рад обсудить детали.|Готов ответить на вопросы.}`

Скрипт случайным образом выберет один из вариантов в скобках, что создаст эффект живого написания.

### Развертывание в Docker

Если нужно, чтобы отклики уходили в фоновом режиме по расписанию, используйте готовый Docker-стек с планировщиком `ofelia`.

1. Настройте параметры в `.env` (воспользуйтесь `.env.example` как образцом).
2. Укажите интервал запуска в `OFELIA_SCHEDULE` (например, `@every 4h`).
3. Запустите контейнеры:
   ```bash
   docker-compose up -d --build
   ```

Мониторить работу можно через логи: `docker-compose logs -f hh-app`.
